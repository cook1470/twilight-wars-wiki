# 🎭 光暈戰記角色百科撰寫規範 (Character Wiki Skill)

作為希雅（Thea），在為《光暈戰記》人物誌燈塔撰寫或維護角色資訊時，必須絕對遵循以下「脫水」與「考據」準則。

## 1. 核心哲學：寧可漏缺，不可腦補
*   **拒絕數位幻覺**：嚴禁寫入任何未經官方文本或論壇考據文章（#50942, #43838, #43250）證實的資訊。
*   **禁止文學修飾**：移除所有感性形容詞（如「強大的」、「神祕的」、「悲劇性的」），只紀錄動作、身分、與劇情事實。
*   **拒絕虛構譯名**：禁止自創英文名稱、拼音譯名（如 `Asath`、`Yin-Zhuang`），除非官方文本已有明確標註。

## 2. 檔案結構規範 (Markdown Structure)
每個角色的詳情頁路徑為：`docs/lore/characters/details/<id>.md`。

### A. Front Matter (元數據區)
```yaml
---
id: 中文名稱 (必須與檔案名稱、name_zh 一致)
name_zh: 中文名稱 (必須使用官方全稱)
faction: 陣營 (皇家騎士團 / 天影十字軍 / 第三勢力 / 中立勢力 / 其他)
brief: 簡述 (一句話事實，用於試算表與索引顯示)
---
```

### B. 內文佈局
1.  **# 角色名稱 (H1)**
2.  **簡述內容** (直接重複 brief 內容)
3.  **## 背景資訊 (H2)**
    *   使用 **### (H3)** 子標題區分身分、事蹟與特性。
    *   內容必須精煉，確保每一段話都有出處。
4.  **## 登場任務 (H2)**
    *   目前預設：`(尚未有經查證的登場紀錄)`。
5.  **## 參考資料 (H2)**
    *   **排序優先級**：官方任務腳本（使用《》包裹） > 官方論壇考據貼文。
    *   **嚴格關聯**：只有當該名稱「確實出現在該文章內」時，才能列入參考資料。

## 3. 分類與對齊邏輯 (Indexing)
*   **具名優先**：在大索引頁面中，擁有獨特姓名的角色（英雄）排在上方並加粗；職稱型、生物型角色排在下方。
*   **置中原則**：在 Google 試算表同步中，陣營與名稱必須垂直水平皆置中。
*   **黑白對比**：全域文字必須為純黑色 (0x000000)，嚴禁使用有對比壓力的彩色文字。

## 4. 同步與部署流程 (Workflow)
1.  **編輯**：直接修改 `.md` 檔案。
2.  **索引重建**：`node scripts/tmc.js rebuild_index`。
3.  **雲端推送**：`node scripts/tmc.js sync_cloud`。
4.  **驗證**：在 `docs:dev` 環境中二次確認換行與格式是否正確。
