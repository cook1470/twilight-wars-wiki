# 🎭 角色百科添加指南 (Character Addition Guide)

為了維持 Wiki 內容的一致性與專業感，未來添加或編輯角色資訊時，請希雅（或任何管理代理人）嚴格遵守以下格式規範。

## 1. 檔案結構 (Markdown Structure)
每個角色的檔案路徑應為：`docs/lore/characters/details/<id>.md`。

```markdown
---
id: 角色唯一識別碼 (小寫英文底線)
name_zh: 中文全名 (含特殊符號如：阿波羅．哈斯卡)
faction: 所屬陣營 (天影十字軍 / 皇家騎士團 / 第三勢力 / 其他)
brief: 角色簡述 (一句話，約 30-50 字，用於試算表與索引顯示)
---

# 角色名稱 (H1)

簡述內容 (重複 Front Matter 中的 brief)

## 背景資訊 (H2)

### 子標題 (H3)
詳細的背景描述、生平、性格、能力設定等。

## 登場任務 (H2)
[任務名稱 A](連結)、[任務名稱 B](連結)
(若尚未查證，請填寫：尚未有經查證的登場紀錄)

## 參考資料 (H2)
- [《光暈戰記》官方任務：【任務名稱】](https://twilightwars.gamelet.online/)
- [外部連結標題](URL)
```

## 2. 撰寫準則 (Content Rules)

### 🚫 嚴禁行為 (Forbidden)
*   **嚴禁腦補**：不可推測角色的內心戲、戰鬥數值或 AI 行為，除非官方文本有明確描述。
*   **嚴禁虛構譯名**：不可自創英文拼音或譯名。
*   **隱藏系統代碼**：在 `#` 標題、`##` 內容區，嚴禁出現 `.events` 等後端代碼。

### ✅ 必須執行 (Required)
*   **極簡登場任務**：登場任務區塊僅包含「任務名稱」與「超連結」，多個任務間使用「頓號（、）」串接，不換行，不加括號。
*   **陣營優先級**：全專案遵循 **天影十字軍 > 皇家騎士團 > 第三勢力** 的順序。
*   **純淨化處理**：寫入時必須過濾掉 `\n`、`\t` 等轉義字元。
*   **優先順序**：參考資料區塊中，「官方任務」與「陣營背景」必須排在所有「論壇連結」之前。

## 3. 同步工作流 (Sync Workflow)
添加完 `.md` 檔案後，必須執行以下指令：

1.  **`node scripts/rebuild-character-indexes.js`**：重建陣營索引頁。
2.  **`node scripts/sync-md-to-sheet.js`**：同步至 Google 試算表。
