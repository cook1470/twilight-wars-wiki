# 🎭 角色資料重建協議 (Character Reconstruction Protocol)

此文件定義了希雅 (Thea) 在補全《光暈戰記》角色百科時必須遵循的標準作業程序 (SOP) 與撰寫規範。核心原則為「官方文本優先，嚴禁主觀腦補」。

## 🛠️ 執行步驟 (Workflow)

### 1. 官方任務深度掃描 (真理來源)
*   **動作**：對 `docs/missions/` 下的各陣營任務進行全文檢索（grep）。
*   **提取重點**：角色在劇情對話中展現的性格、動機或背景故事。專注於「靈魂」而非「戰鬥數值」。
*   **別名處理**：若無結果，應嘗試搜尋別名或職稱（如「天影師父」），並重複此步驟。

### 2. 論壇數據查證 (輔助資料)
*   **參考檔案**：`docs/lore/characters/RAW_NAMES.txt` 或官方論壇考據貼文。
*   **提取重點**：精確的登場關卡、模型名稱（如：薔薇刺客）。

### 3. 提交草稿審閱 (關鍵防線)
*   **動作**：在 Discord 中提出「待錄入草稿」。只有 Cook 確認「OK」後，才能執行錄入。

### 4. 資料庫錄入與同步
*   **工具**：使用 `skills/characters/update/` 與 `skills/wiki/sync/` 底下的最新腳本。
*   **狀態更新**：錄入完成後，確保試算表欄位設為 `TRUE`。

## 📜 撰寫規範 (Writing Specifications)

### A. 檔案結構 (Markdown Layout)
每個角色的檔案應包含以下結構，層級必須清晰：

```markdown
---
id: 角色中文名稱 (必須與 name_zh 一致)
name_zh: 中文全名 (必須使用官方全稱)
name_en: 官方英文名 (若無則留空，嚴禁自創)
faction: 所屬陣營 (天影十字軍 / 皇家騎士團 / 第三勢力 / 中立勢力 / 其他)
species: 角色種類 (人類 / 殭屍 / 靈體 等)
brief: 角色簡述 (一句話事實，約 30-50 字)
---

# 角色名稱 (H1)

簡述內容 (直接重複 Front Matter 中的 brief)

## 背景資訊 (H2)

### 子標題 (H3)
以分段方式撰寫詳細背景、生平與性格描述。段落間使用雙換行 (`\n\n`)。

## 登場任務 (H2)
[任務名稱 A](連結)、[任務名稱 B](連結)
(格式：`[名稱](<絕對路徑>)`，多個任務使用頓號「、」串接，不換行。)

## 參考資料 (H2)
- [《任務名稱》官方任務腳本](URL)
- [官方論壇貼文 #ID](URL)
```

### B. 核心鋼鐵律 (Steel Laws)
1.  **劇本至上**：紀錄動作、身分與劇情事實。移除感性形容詞（如「強大的」、「悲劇性的」）。
2.  **嚴禁腦補**：禁止使用「推測其為...」、「可能因為...」等虛構幻覺字眼。
3.  **拒絕虛構譯名**：嚴禁自創英文名稱或拼音譯名。
4.  **數據純淨**：試算表與 Markdown 內嚴禁行首縮進，寫入時須過濾轉義字元。
5.  **排序優先級**：參考資料中，「官方任務」必須排在「論壇連結」之前。
